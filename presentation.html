Project Overview

프로젝트 개요

미술관 전시 공간을 일반 관람객이 온라인으로 예약하고,
관리자가 갤러리·예약·업무를 한 곳에서 운영할 수 있는
풀스택 웹 예약 관리 플랫폼입니다.

👤

일반 사용자

갤러리 탐색, 날짜·인원 선택 후 예약, 예약 내역 조회·취소

🛠️

관리자

갤러리 등록·수정·삭제, 예약 전체 조회, 운영 할 일 관리

🔒

보안

Troubleshooting

주요 트러블슈팅

①

전화번호 필드에 문자열 입력 시 500 에러

원인: 예약 폼의 전화번호 필드 타입이 int (또는 Integer)로 선언되어 있어, 문자열 입력값이 바인딩될 때 MethodArgumentTypeMismatchException 발생 → 500 응답.
해결: 전화번호 필드 타입을 String으로 변경 후 서비스 계층에서 정규식(^\d{10,11}$)으로 유효성 검증, 오류 시 사용자에게 안내 메시지 반환.

②

로그인 직후 403 에러 (CSRF 쿠키 누락)

원인: Spring Security 6 lazy CSRF loading — 긴 HTML 렌더링 중 응답 버퍼 flush 시 Set-Cookie 헤더 누락.
해결: CsrfCookieFilter (OncePerRequestFilter)로 렌더링 전 csrfToken.getToken() 강제 호출.

③

갤러리 수용 인원 초과 예약 신청 시 오류 미처리

원인: 예약 인원 입력값이 갤러리의 capacity를 초과해도 서버에서 별도 검증 없이 저장되거나, DB 제약 조건에서 예외가 발생해 사용자에게 500 에러 페이지가 노출됨.
해결: Service 계층에서 requestedCount > gallery.getCapacity() 조건 검증 추가, 초과 시 IllegalArgumentException을 던지고 Controller에서 잡아 폼에 errorMessage로 "수용 인원(N명)을 초과할 수 없습니다." 안내.
